# MVP 需求文件

## 1. 專案目標

本專案是一個 **多人協作自由行規劃網站**，目標是讓一群人可以共同規劃同一趟旅程，同時保留每個人各自的行程安排。

核心目標如下：

- 任何註冊使用者都可以建立旅程
- 建立旅程的人為該旅程的團主
- 團主可以邀請其他成員加入旅程
- 每個成員都可以擁有自己的個人行程
- 某一天或某個時段可以設定為共同行程
- 使用者可以參考別人的行程，並複製別人的某一天到自己的個人行程
- 網站需同時支援手機與電腦使用，且手機體驗需優先考量

---

## 2. 使用者角色與權限

### 2.1 團主
團主為旅程建立者，具備以下權限：

- 建立旅程
- 管理旅程基本設定
- 邀請成員加入旅程
- 管理成員角色
- 設定共同行程
- 管理住宿資訊
- 查看所有成員行程
- 查看全員總表

### 2.2 成員
成員為參與旅程的一般使用者，具備以下權限：

- 查看旅程資訊
- 查看其他成員行程
- 建立與編輯自己的個人行程
- 將其他成員的某一天行程複製到自己的個人行程
- 查看共同行程與全員總表
- 在團主授權的情況下，可建立、編輯與刪除共同行程
  
### 2.3 只讀成員
只讀成員僅能查看資料，不可進行編輯，具備以下權限：

- 查看旅程資訊
- 查看其他成員行程
- 查看共同行程
- 查看全員總表

### 2.4 權限原則
- 成員只能編輯自己的個人行程
- 只讀成員不可編輯任何資料
- 團主是否可直接編輯他人的個人行程，暫列為待確認事項
- 成員是否可設定共同行程，取決於團主是否授權

---

## 3. 行程結構

### 3.1 行程類型
同一趟旅程中，行程分為兩種：

#### 個人行程
- 每位成員都有自己的個人行程
- 個人行程由本人建立與編輯
- 其他成員可查看，但不可修改

#### 共同行程
- 團主可直接設定；成員需在獲得團主授權後才可設定
- 可指定某一天或某個時段為共同活動
- 共同行程會顯示在全員總表中

### 3.2 行程混合模式
- 同一天可同時存在 **共同行程** 與 **個人自由行程**
- 不要求整天只能屬於單一模式
- 例如上午可為共同行程，下午則為個人自由安排

### 3.3 行程複製
- 成員可複製其他成員的某一天行程到自己的個人行程
- MVP 第一版先支援 **整天複製**
- 部分項目複製可列為後續優化功能

---

## 4. 手機版與電腦版 UI 原則

### 4.1 手機版原則
手機版為優先設計對象，重點如下：

- 使用者先選擇日期
- 選定日期後，顯示所有成員的卡片式列表
- 每張卡片顯示該成員當天的行程摘要
- 使用者可快速查看自己的行程
- 使用者可快速查看別人的行程
- 使用者可快速複製別人的某一天行程到自己這裡
- 手機版以方便查看、討論、臨時調整為主

### 4.2 電腦版原則
電腦版可提供更完整的資訊呈現方式，重點如下：

- 可使用時間軸模式顯示行程
- 可使用總表模式查看全員行程
- 可同時呈現較多資訊，方便整體比較與討論

---

## 5. MVP 必做功能

MVP 第一版必須包含以下功能：

1. 建立旅程
2. 邀請成員加入旅程
3. 每位成員建立與編輯自己的個人行程
4. 查看其他成員的行程
5. 顯示全員總表
6. 設定某一天或某個時段為共同活動
7. 複製其他成員的某一天行程到自己的個人行程
8. 紀錄住宿基本資訊

---

## 6. 第二版功能

以下功能不納入 MVP，規劃於第二版再評估實作：

- 留言功能
- 預算追蹤
- 通知系統
- Google Maps 整合
- 路線推薦
- 天氣 API 整合

---

## 7. 住宿資料需求

MVP 第一版的住宿資料欄位如下：

- 名稱
- 地址
- 入住日期
- 退房日期
- 備註
- 房間資訊（可選）

說明：
- 第一版先支援基本住宿資訊紀錄
- 第一版不強制整合外部住宿 API

---

## 8. 待確認事項

以下需求仍需後續確認：

- 團主是否可直接編輯他人的個人行程
- 複製行程後，是否需要標示來源成員
- 未來是否需要支援部分項目複製，而不只整天複製
- 第二版的 Google Maps、天氣 API、路線推薦要整合到什麼程度
